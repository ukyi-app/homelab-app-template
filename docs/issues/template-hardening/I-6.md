# I-6 — Renovate 설정 동봉

## Parent

`docs/prds/template-hardening.md`

## What to build

스캐폴드된 앱이 Renovate 설정을 상속해, **homelab 쪽 레포 등록만 되면 즉시**
base 이미지·의존성 보안 패치가 흐르기 시작할 준비를 갖추게 한다. 현재 renovate
설정은 템플릿 루트에만 있어 생성된 앱에 복사되지 않으며, 그 결과 생성된 앱의
base 이미지·의존성 핀이 영구히 정체된다.

공통 스캐폴드 자산에 renovate 설정을 추가한다 — 템플릿 루트 설정과 같은 정책
(권장 프리셋 + digest 핀)을 따른다.

**범위 경계**: homelab 쪽 레포 등록 배선(Renovate 실행 대상 목록 추가 또는
allowlist autodiscover 전환)은 교차-레포 작업으로 이 배치의 범위 밖이며,
PRD의 Follow-up Backlog F-1로 추적된다. 이 이슈는 "등록 즉시 동작할 준비"까지만
책임진다.

## Acceptance criteria

- [ ] 4개 아키타입 어느 것으로 스캐폴드해도 앱 루트에 renovate 설정이 존재한다.
- [ ] 설정이 템플릿 루트와 동일한 정책(권장 프리셋 + digest 핀)을 따른다.
- [ ] 스캐폴더가 남기는 파일 목록에 renovate 설정이 포함되고, 스캐폴드 자체는
      계속 성공한다(자가삭제·lock 재생성 경로 영향 없음).
