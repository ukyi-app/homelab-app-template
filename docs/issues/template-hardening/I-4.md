# I-4 — 스캐폴더 강화: 앱 식별자·플래그 경고·주석 정직화

## Parent

`docs/prds/template-hardening.md`

## What to build

스캐폴더가 배포 식별자 계약(ADR-0002: 앱 이름 = GitHub 레포명)과 어긋난 산출물을
조용히 만들어내지 않게 한다.

- **앱 이름 유도**: 이름 기본값을 git origin URL에서 유도한다(origin이 없으면
  디렉토리명 fallback). 사용자가 명시한 이름이 유도값과 다르면 "이 값은 배포
  식별자가 아니며 GitHub 레포명과 같아야 한다"고 stderr로 **경고**한다 — 하드
  실패시키지 않는다(최종 강제는 homelab create-app에 이미 존재하고, CI가 임시
  디렉토리에서 `--name ci-<archetype>`으로 스캐폴드하는 정당한 사용처가 있다).
- **봉인 산출물 이름 고정**: 생성되는 `secret:seal` 스크립트에 유도된 앱 이름을
  `--app`으로 베이크해, 봉인 산출물 이름이 실행 디렉토리명이 아니라 앱 이름에서
  파생되게 한다(현재는 cwd basename에 의존해 클론 디렉토리명이 다르면 create-app
  온보딩 검사에서 리젝될 수 있다).
- **비호환 플래그 경고**: kind와 무관한 플래그(worker에 public/metrics, site에
  metrics)가 비대화형에서 주어지면 조용히 드롭하지 말고 stderr 경고 후 무시한다.
  알 수 없는 플래그는 기존대로 즉시 실패(exit 2) — 오타와 의도는 다르게 다룬다.
- **롤백 주석 정직화**: 스캐폴더의 "재시도 가능" 주석을 실제 동작(실패 시
  `git checkout .`으로 복구)에 맞게 정정한다. 롤백 메커니즘 자체는 손대지 않는다.

## Acceptance criteria

- [ ] git origin이 있는 레포에서 스캐폴드하면 앱 이름 기본값이 origin의 레포명에서
      유도된다. origin이 없으면 디렉토리명으로 fallback한다.
- [ ] 명시한 이름이 유도값과 다르면 stderr 경고가 나오되 스캐폴드는 성공한다
      (CI의 `--name ci-<archetype>` 사용처가 계속 동작한다).
- [ ] 생성된 앱의 `secret:seal` 스크립트가 `--app <앱 이름>`을 포함해, 다른 이름의
      디렉토리에서 실행해도 봉인 산출물 이름이 앱 이름 기준으로 나온다.
- [ ] `--archetype worker --public`, `--archetype worker --metrics`,
      `--archetype site --metrics`가 각각 stderr 경고를 내고 무시된다(성공 종료).
- [ ] 알 수 없는 플래그는 여전히 exit 2로 거부된다(기존 네거티브 테스트 통과).
- [ ] 스캐폴더의 롤백 주석이 실제 복구 절차를 정직하게 기술한다.
- [ ] 4개 아키타입 비대화형 스캐폴드가 여전히 성공한다.
