# I-1 — graceful shutdown: 런타임 종료 계약 완성

## Parent

`docs/prds/template-hardening.md`

## What to build

스캐폴드되는 모든 아키타입이 SIGTERM에 유예시간 내 정상 종료하도록 만든다.
distroless 이미지에서 앱 프로세스는 PID 1로 뜨므로, 명시 핸들러가 없는 시그널은
커널이 기본 처리(terminate)하지 않고 무시한다 — 핸들러 등록 자체가 런타임 계약의
일부다.

- **web 계열(api·fullstack)**: 메인 서버와 metrics 서버의 핸들을 붙잡아, SIGTERM과
  SIGINT 어느 쪽이 와도 두 서버를 모두 정지시킨 뒤 종료 코드 0으로 끝낸다. 드레인
  세리머니(preStop 대기 등)는 넣지 않는다 — 차트 영역이다.
- **worker**: 현재 SIGTERM은 플래그만 내리고 루프가 sleep에 갇혀 남은 주기를 다
  기다린다. sleep을 **중단 가능**하게 만들어 시그널 수신 즉시 깨어나게 하고, 종료
  로그를 남기고 코드 0으로 끝낸다. 또한 루프 본문(작업 자리)을 예외 격리해 일시
  오류가 로깅·백오프로 흡수되고 프로세스를 즉사시키지 않게 한다. 치명/일시 오류를
  구분하는 분류 체계는 **도입하지 않는다**(PRD P-2 결정 — TODO 자리 루프에 과설계).

## Acceptance criteria

- [ ] api·fullstack: SIGTERM 수신 시 메인 서버와 metrics 서버가 모두 정지하고
      프로세스가 종료 코드 0으로 끝난다 (SIGINT도 동일 경로).
- [ ] api·fullstack: 종료가 유예시간(30초)보다 훨씬 짧게, 수 초 이내에 완료된다.
- [ ] worker: sleep 도중 SIGTERM이 도착하면 남은 sleep을 기다리지 않고 즉시
      깨어나 종료 로그 후 코드 0으로 끝난다.
- [ ] worker: 루프 본문에서 발생한 예외가 프로세스를 죽이지 않고 로깅·백오프로
      흡수된다.
- [ ] 4개 아키타입 모두 기존 typecheck(`bunx tsc --noEmit`)와 빌드를 통과한다.

## Blocked by

None - can start immediately.
