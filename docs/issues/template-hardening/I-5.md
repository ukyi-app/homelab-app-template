# I-5 — DX/문서: dev 스크립트·.env.example·README 정합

## Parent

`docs/prds/template-hardening.md`

## What to build

스캐폴드된 앱의 첫 개발 경험과 문서가 실제 동작과 일치하게 만든다.

- **fullstack 통합 dev**: 기본 추천 아키타입인 fullstack에 README가 안내하는
  `dev` 스크립트가 없다(dev:web·dev:server만 존재). concurrently를 devDependency로
  추가해 서버와 웹 개발 프로세스를 한 명령으로 병렬 기동한다(접두어 로그, 한쪽이
  죽으면 동반 종료).
- **비밀 값 흐름 문서화**: `.env.example`을 공통 스캐폴드 자산으로 동봉한다 —
  키 명명 규약(UPPER_SNAKE)과 DB URL 접미사 규약(`<APP>_DATABASE_URL` /
  `_MIGRATE_` / `_RO_`, generic fallback)을 주석으로 담는다. 앱 README에 '비밀 값'
  섹션을 신설해 .env 작성 → 봉인 실행(kubeseal 필요) → 봉인 산출물 커밋의 3단계를
  명시한다.
- **README 런타임 문구 정합**: 앱 README의 런타임 한 줄이 4개 아키타입에 일괄로
  "arm64 distroless non-root, http :8080"을 주장하는데, worker는 HTTP를 서빙하지
  않고 site는 distroless가 아니라 정적 서버 이미지다. 스캐폴더가 README를 이미
  후처리하므로, 런타임 문구를 **아키타입별로 치환**하게 한다.
- **개발 섹션**: fullstack의 2-프로세스 구조(웹 dev 서버가 API를 프록시)를 README
  개발 섹션에 설명한다.

## Acceptance criteria

- [ ] fullstack으로 스캐폴드한 앱에서 `bun run dev` 한 명령이 서버와 웹 dev 서버를
      함께 기동한다(concurrently가 devDependency에 포함된다).
- [ ] 스캐폴드된 앱 루트에 `.env.example`이 존재하고, UPPER_SNAKE 규약과 DB URL
      접미사 규약이 주석으로 설명되어 있다.
- [ ] 앱 README에 '비밀 값' 섹션이 있고 .env 작성 → 봉인 → 산출물 커밋 3단계와
      kubeseal 필요성이 명시된다.
- [ ] 앱 README의 런타임 문구가 아키타입별로 정확하다: worker는 HTTP 비서빙,
      site는 정적 서버(distroless 아님), api·fullstack은 :8080 distroless.
- [ ] 앱 README 개발 섹션이 해당 아키타입에서 실제로 동작하는 명령만 안내한다
      (fullstack의 2-프로세스·API 프록시 구조 포함).
- [ ] 4개 아키타입 비대화형 스캐폴드가 성공하고 기존 계약 검증이 통과한다.
